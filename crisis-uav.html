Crisis-Courier-UAV

Designing the human interface for an autonomous UAV supply delivery system in disaster zones — where cognitive overload isn't an inconvenience, it's a mission failure.

Role Human Systems Designer
Timeline Aug – Dec 2024
Domain Disaster Relief / UAV Systems
Context Academic Research
Crisis-Courier-UAV — Ground Control Interface

Overview

When the interface fails, the mission fails — designing for zero errors under pressure

Crisis-Courier-UAV is an autonomous drone system designed to deliver medical supplies, food, and communications equipment into disaster zones inaccessible to ground vehicles. The human interface — the ground control station used by relief coordinators to manage fleet operations — is where human judgment meets autonomous systems.

This project applied human factors and safety-critical design principles to the ground control interface. The core design challenge: operators are already under extreme cognitive load from the disaster context. The interface cannot add to that load — it must actively reduce it.

52% reduction in operator cognitive load (NASA-TLX)
3× more UAVs manageable per operator vs. baseline system
0 critical errors in simulated disaster scenario testing

The Challenge

Extreme constraints — designing for the worst possible conditions

Disaster zone operation means the operators using this interface are working under conditions that no normal UX research accounts for: physical exhaustion, emotional stress, sleep deprivation, partial information, and time pressure measured in minutes, not hours.

Standard usability principles — while necessary — aren't sufficient. The design had to draw on aviation human factors, military interface standards (MIL-STD-1472), and emergency dispatch system research to establish the right baseline.

Cognitive Overload
Operators managing multiple simultaneous UAV missions while coordinating with field teams via radio and tracking supply inventory.
Environmental Noise
Field stations may be improvised — poor lighting, generator noise, damaged equipment. Interface must function at 50% visual attention.
Intermittent Connectivity
Satellite uplink may be degraded or interrupted. Interface must handle partial data and predict UAV state between updates.
Non-Expert Operators
Relief coordinators are domain experts (logistics, medicine) but not drone pilots. Interface must be learnable in under 2 hours.
Zero Tolerance for Error
A misrouted UAV carrying insulin to the wrong location isn't a UX problem — it's a life-safety incident. Error prevention is existential.
Autonomy Calibration
Too much automation: operators lose situational awareness. Too little: they're overwhelmed. The interface must find the right human-automation balance.

"In a disaster, every decision costs something. The interface can't make me think harder — it has to think with me."

— Emergency logistics coordinator, user research session

Process

Human factors methodology — not standard UX practice

This project required stepping beyond typical product design research. I applied a human factors framework: cognitive task analysis (CTA) to understand what operators actually need to think about, workload assessment using NASA-TLX, and simulation-based testing with disaster scenario scripts.

01
Cognitive Task Analysis
Mapped every decision an operator needs to make during active UAV operations. Identified 14 concurrent information needs during peak operations.
02
Literature Review
Aviation HCI, military interface standards (MIL-STD-1472), emergency dispatch research. Established design principles from validated safety-critical systems.
03
Design
Ground control interface with 3 information zones: tactical map (where), fleet status (what), and action queue (what next). Designed for glanceable comprehension.
04
Simulation Testing
4 simulated disaster scenarios with NASA-TLX workload assessment. Compared against baseline interface. Iterated until cognitive load metrics improved by target threshold.

Solution

Situational awareness by design — the right information, always visible

The ground control interface is organized around the concept of situational awareness zones: a tactical map at center (geographic reality), fleet status panels at left (UAV states at a glance), and an action queue at right (what decisions need to be made, in priority order). Nothing is hidden behind tabs during active operations.

  • Glanceable fleet status: Each UAV is a single card showing battery, payload, status, and ETA. Designed to be understood in under 3 seconds without reading.
  • Priority-based action queue: The interface surfaces decisions that need human input, ranked by urgency. Operators respond to the queue — they don't have to monitor everything constantly.
  • Alert hierarchy: Three levels of alert — informational (blue), attention (amber), critical (red). Critical alerts include a mandatory acknowledgment step. No alert fatigue from overuse of red.
  • Autonomy dial: Per-UAV automation level (1–5) lets operators increase human control for sensitive missions (medical delivery to specific recipient) and reduce it for bulk supply runs.
  • Predictive state display: During connectivity loss, the interface shows last-known state + predicted current position based on flight plan. Prevents false emergency responses.
  • 2-hour onboarding: Role-based interface defaults surface only the features relevant to the operator's task. Advanced controls are present but de-emphasized until needed.

Results

Lower load, higher capacity, zero critical errors

Simulation testing against a baseline interface showed substantial improvements across all measured dimensions. The 52% reduction in NASA-TLX cognitive load score was the primary target metric. The increase in manageable UAVs per operator — from 2 to 6 — has direct operational impact on how many people can be reached in a disaster scenario.

52% reduction in NASA-TLX cognitive load score
3× more UAVs managed per operator (2 → 6)
0 critical mission errors across all simulation runs
